
## 1. Що таке `int 0x80`

`int 0x80` — це інструкція програмного переривання в архітектурі x86, яка використовується у 32-бітному Linux для виконання системних викликів (system calls).

Після виконання цієї інструкції процесор переходить у режим ядра, передає керування обробнику переривання №128 (0x80) і виконує відповідний системний виклик.

Механізм роботи:

1. У регістр `eax` записується номер системного виклику.
2. У регістри `ebx`, `ecx`, `edx`, `esi`, `edi`, `ebp` передаються аргументи.
3. Виконується `int 0x80`.
4. Результат повертається в `eax`.

---

## 2. Найбільш часто використовувані системні иклики

Нижче наведені популярні системні виклики для 32-бітного Linux.

### sys_exit (номер 1)

Завершення процесу.

Параметри:

* `ebx` — код завершення

Приклад:

```asm
mov eax, 1      ; sys_exit
mov ebx, 0      ; status = 0
int 0x80
```

---

### sys_write (номер 4)

Запис у файл або stdout.

Параметри:

* `ebx` — дескриптор файлу
* `ecx` — адреса буфера
* `edx` — кількість байтів

Приклад:

```asm
mov eax, 4          ; sys_write
mov ebx, 1          ; stdout
mov ecx, message    ; адреса рядка
mov edx, length     ; довжина
int 0x80
```

---

### sys_read (номер 3)

Читання з файлу або stdin.

Параметри:

* `ebx` — дескриптор файлу
* `ecx` — адреса буфера
* `edx` — максимальна кількість байтів

---

### sys_open (номер 5)

Відкриття файлу.

Параметри:

* `ebx` — шлях до файлу
* `ecx` — прапори відкриття
* `edx` — права доступу (mode)

---

### sys_close (номер 6)

Закриття файлу.

Параметри:

* `ebx` — дескриптор файлу

---

## 3. Зауваження

* `int 0x80` використовується у 32-бітному Linux.
* У 64-бітних системах застосовується інструкція `syscall`.
* `int 0x80` повільніший за `syscall` та `sysenter`, оскільки використовує механізм програмного переривання.

---

## Висновок

`int 0x80` — це класичний механізм виконання системних викликів у 32-бітному Linux через програмне переривання 0x80. Він дозволяє користувацькому процесу взаємодіяти з ядром операційної системи.
